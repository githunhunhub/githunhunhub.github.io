用户信息

在安装完Git后首先要做的事：设置用户名称与邮件地址。之后每个Git的提交都会用到这些信息，并且写入到每次的提交中，不可更改。

$ git config --global user.name "xxxx"
$ git config --global user.email xxxxx@example.com

强调：使用 --global 选项，那么该命令只用运行一次，之后所有仓库提交都会用这个信息，若要单独使用不同用户名和邮箱，可在相应项目下运行没有 --global 选项的命令配置。

创建 SSH Key
  1. 检查目录 C:\Users\用户名\.ssh 下有没有 id_rsa 和 id_rsa.pub 这两个文件，若有则已经有key了。
  2. 创建 SSH kye:
    $ ssh-keygen -t rsa -C "xxxxx@example.com"
  3. 登陆GitHub，打开“Account settings” , 在SSH Keys页面添加key, 把id_res.pub(可用记事本打开)里边的内容复制进去。
创建仓库
  1. 初始化： git init
  2. 关联远程仓库：git remote add origin git@github.com:yourGitHubname/repositoryname.git
同步到仓库
  1. 先检查文件状态：git status
  2. 将修改：git add --all 或是 git add .
  3. 再检查：git status
  4. 提交: gti commit -m "xxxxxxxx" XX处填写此次提交的一些信息
  5. 新仓库第一次推送：git push -u origin master
  6. 之后的推送：git push origin master

注意： 第一次使用Git的 push 和 clone 连接GitHub时，会出现警告，输入yes即可：

The authenticity of host 'github.com (xx.xx.xx.xx)' can't be established.
RSA key fingerprint is xx.xx.xx.xx.xx.
Are you sure you want to continue connecting (yes/no)?
